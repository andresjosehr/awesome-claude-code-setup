# Actualizar Documentacion

Por favor, actualiza la documentacion del proyecto para reflejar cambios recientes:

1. **Identificar cambios recientes** - Revisar historial de git y archivos modificados para entender que ha cambiado

2. **Actualizar README.md** - Asegurar que el README principal refleja:
   - Cualquier nueva funcionalidad o capacidad
   - Instrucciones de configuracion actualizadas si es necesario
   - Nuevas variables de entorno
   - Puertos o servicios cambiados

3. **Actualizar CLAUDE.md** - Agregar cualquier:
   - Decisiones arquitectonicas nuevas
   - Patrones de desarrollo descubiertos
   - Problemas comunes y soluciones
   - Percepciones de pruebas

4. **Revisar docs de componentes** - Actualizar documentacion para:
   - Nuevas APIs o endpoints
   - Flujos de trabajo modificados
   - Esquemas de datos cambiados
   - Nuevas integraciones

5. **Actualizar documentacion en linea**:
   - Comentarios JSDoc/TSDoc para nuevas funciones
   - Comentarios en archivos de configuracion
   - Descripciones de variables de entorno

6. **Verificar precision** - Asegurar que todos:
   - Los ejemplos de codigo aun funcionan
   - Los comandos estan actualizados
   - Los enlaces no estan rotos
   - Los numeros de version son actuales

Enfocate en la claridad y en ayudar a futuros desarrolladores (incluyendote a ti mismo) a entender el sistema rapidamente.
